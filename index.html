语音朗读功能

小说插件9.7.1版本起，语音朗读技术更换为百度语音，因讯飞关闭了html5平台的文字转语音接口
插件中已内置百度语音技术，可朗读章节正文，要使用此功能，必须先申请百度语音的 API Key 和 Secret Key，并填写到插件设置页面的 百度语音API Key 和 百度语音Secret Key 选项中

默认填写的是一个测试用api key和 secret key。接口调用次数受限制，为了正常使用，你需要去申请自己的来进行替换。

申请方法

申请页面网址
http://yuyin.baidu.com

如果你有百度帐号，请点击登录，如果没有，请注册。成功登录后，再点击此 应用管理 按钮

单击 创建新应用，会弹出如下提示，然后输入应用名称和选择应用类型

然后点击下一步，这一步中，必须选中 语音合成 ，否则无法使用

继续点击下一步，这一步，什么都不用选，继续点击下一步。

这里选中 android 类型，包名填写你的域名的倒写形式，如 com.baidu.www ，然后点击下一步

到这里就创建成功了，点击服务器设置关闭此弹层即可

复制 api key 和 secret key

点击应用管理，找到你刚刚创建的应用，点击 查看key

复制 api key 粘贴到 插件设置页面的 【百度语音api key】中

大功告成，就这么简单，在章节页面，如果用户具有阅读本章节的权限，就会看到朗读按钮，点击即可朗读了

伪静态

只有在你的论坛当前已在正常使用伪静态的情况下，才可使用插件伪静态

如果确定你的服务器支持伪静态，请复制对应你服务器的伪静态规则到你当前可用的规则底部，然后到插件后台-设置–启用伪静态，选择【是】，提交，更新缓存。

目前只提供apache、IIS6、IIS7、nginx服务器下的伪静态规则，其他服务器请根据apache的自行改写

WebApp

webapp 由html文件和js文件组成，可放在任意可从浏览器访问到的位置，可使用论坛帐号登录和在webapp中注册，适合作为小说插件的手机端替换或者单独打包为app

一：购买 webapp源码和接口文件

源码由两部分组成，webapp文件夹+api.php文件

  • webapp文件夹中的所有文件放在你网站的任意目录下，比如放在 app目录下，则可通过 你的网址/app 进行访问，如果直接放在网站根目录下，则可直接通过 你的网址 进行访问
  • api.php 文件放在你安装有 小说阅读器 专业版插件的网站根目录下的 xifeng 文件夹中，如果不存在这个文件夹，则手动创建

二：修改配置

  • 打开 源码中 book文件夹下的faxian.html文件，找到 http://xf512.com，替换为你的网址，这里的网址是你的官网，建议链接至你的论坛
  • 打开 源码中 script文件夹下的api.js文件，找到 http://demo.xf512.com/xifeng/ 替换为 你放置api.php文件的网址，比如你放在了网站 www.abc.com/xifeng 下，则这里填写 http://www.abc.com/xifeng/

三：添加幻灯、公告、专题等

进入网站后台-应用-小说阅读器-手机版-拉到底部，填写app相关的选项

做好这些后，布置就完成了，去 网站后台-应用-小说阅读器-手机版–拉到底部–app相关的选项填写上即可。 接下来你就可以将地址告知你的用户或者去打包了

插件设置选项

网站后台->应用->小说阅读器->设置

本插件中设置选项较多，如下图，将在这里介绍用途

  1. 本插件显示名称:这里设置的文字将显示在 某些需要链接到插件首页的地方，比如阅读页面到首页的链接字样
  2. 图书存储的板块:必须选择一个板块，章节数据将存储在这个板块中，设置后，这个板块将自动设为隐藏，前台不可见，同时如果你想在章节中使用html代码和discuz以及图片、视频等，在必须再到 后台-论坛-这个板块-编辑-帖子选项-允许html+允许使用discuz代码+允许使用多媒体+允许解析图片中都选择是
  3. 必须登录才可阅读：设为【是】，则即使是免费章节，访客也必须登录才可阅读，选择【否】，则访客无需登录就能阅读免费章节
  4. 图书默认封面：如果在创建图书时，没有手动添加图书封面，则使用这里填写的图片作为封面，默认封面是 ./source/plugin/jameson_read/images/book.jpg,你可以修改成其他的图片地址，比如http://xf512.com/xifeng/demo.png
  5. 图书热门标签:这里定义的标签将显示在标签页面右侧，一行一个标签
  6. 阅读页面样式：此处有四个可供选择的值，对应着4种不同的阅读样式风格
  7. 阅读页禁止右键：选择 【是】后，在阅读页面将禁止使用鼠标右键
  8. 允许发布图书用户组：可多选，选中的用户组中的用户将自动成为作者，可发布图书
  9. 允许设置售价的用户组：只有在选中的用户组中的用户才可设置图书售价
    13. 允许设置整书售价：此选项选【是】，则作者可为图书设置一个总价，打包销售，否则只能按章节销售
  10. 单本书最高售价：作者设置的最高图书价格，单位是后面设置的图书交易类型
  11. 单章节最高售价：作者设置的单章节最高售价
  12. 图书售价说明：将显示在作者中心给作者解释售价的文字
  13. 收取佣金比率：每笔图书销售、打赏，都将从作者收入中扣取此处设置的百分比作为佣金汇入 uid=1的创始管理员帐号中
  14. VIP用户组|免费阅读：在这些用户组中的用户，可免费阅读收费图书和章节
  15. 免费下载txt用户组：这些用户组中的用户可以免费下载收费图书txt，其他用户必须购买收费图书后才可下载txt
  16. 启用书评功能/启用书评区域:选择【是】，将启用相应功能
  17. 每用户每书书评次/书评最大字数：顾名思义不再解释
  18. 显示话题区域：如果选【是】，则在图书详情页显示绑定的图书板块，并且管理员可在 插件后台–在架图书为每一个图书绑定一个板块，在绑定的板块顶部将显示此图书信息
  19. 图书交易积分类型：可在当前dz后台启用的积分类型中选择，用哪一种作为交易的积分
  20. 打赏用积分类型：打赏专用的积分类型
  21. 分享奖励积分类型/每吸引一IP访问奖励：为鼓励用户分享图书，可在设置每通过分享吸引来一个IP访问奖励的积分类型和数量

  22. PC幻灯图片：显示在PC版首页的幻灯片。

    填写幻灯图片的格式为

    图片地址###跳转网址###图片标题###图片描述|||

    可填写多组，最后一行末尾不要加 |||

  23. PC端分享代码：将显示在阅读界面内容下方的分享按钮，如百度分享

  24. 电脑端充值地址:当用户积分不足时，提示跳转的充值地址，默认是dz自带的充值功能，如果你没有开通，请到网站后台搜索“充值”自行开通
  25. 手机版地址:如果你没有单独的手机域名，请在此填写你的网址即可，以右斜线 / 结尾，如 http://demo.xf512.com/
  26. 手机模板主色调：填写16进制色号，如 #f44336
  27. 手机首页小说中心按钮：将显示在默认手机模板板块列表上方的按钮，点击跳转到小说插件手机版，如果你的手机模板没有显示，说明缺失钩子，请自行添加或者手动加个链接，小说插件手机版地址是 ./plugin.php?id=jameson_read&contrl=touch
  28. 手机端分享代码:阅读页面内容下方的分享代码，使用默认即可
  29. 手机端充值地址：dz没有自带的手机端充值代码，默认将跳转到电脑端充值，如果你有第三方充值插件，在此填写充值地址即可
  30. 作者中心公告/我的书架公告:顾名思义
  31. 讯飞语音APPID/讯飞语音Secret：如果你想使用语音朗读功能，请去讯飞开放平台创建web版语音合成应用，并申请appid来替换此处填写的，申请地址 http://www.xfyun.cn/services/online_tts
  32. SEO关键字/SEO描述：SEO信息
  33. 热搜书名：显示在搜索条中和搜索条旁的默认搜索图书。一行一个书名，第一个将作为搜索框默认值，其他作为热搜词。建议不超过4个，超出显示位置宽度将隐藏
  34. 启用伪静态：只有在你的论坛当前已在正常使用伪静态的情况下，才可启用本项，否则将导致404错误。如果你的论坛已能够正常使用伪静态，还需要根据你的服务器类型，复制对应的下述规则，然后再到此处开启伪静态

#如何将插件设为首页，实现输入网址后自动打开插件首页

discuz系统核心文件限定了移动端首页只能是forum.php，如果设置了其他模块比如插件（plugin.php等）为首页，当移动端上打开插件页面时，dz系统会强制跳转到forum.php即论坛首页面，造成无法在移动端打开插件。

所以如果想将小说中心设为首页，并保证手机端可正常打开，必须按照下述方法修改。
这可能是dz的bug，也可能是dz有意设计。

##有两种方法可以实现

第一种方法：
网站后台-界面-导航设置-小说中心-设为首页，并打开网站根目录下 source/class/discuz/discuz_application.php
找到第778行左右，将 

dheader("location:$mobileurl");

替换为 

// dheader("location:$mobileurl");

优点：简单，无需配置服务器设置
缺点：
1.网址带plugin.php尾巴；
2.此种设置方式对dz手机端无效，手机上首页仍然是forum.php即论坛首页，

第二种方法：
1、仍然需要修改 source/class/discuz/discuz_application.php 这个文件，找到第778行左右，将 

dheader("location:$mobileurl");

替换为 

// dheader("location:$mobileurl");

2、再到你的服务器管理界面中，比如windows上IIS或者Linux管理面板中，在默认文档设置中加一个默认首页 plugin.php ，并移动到第一个位置

3、然后到网站根目录下，打开plugin.php文件，在第23行附近，将

if(!empty($_GET['id'])) {
    list($identifier, $module) = explode(':', $_GET['id']);
    $module = $module !== NULL ? $module : $identifier;
} else {
    showmessage('plugin_nonexistence');
}

替换为

if(!empty($_GET['id'])) {
    list($identifier, $module) = explode(':', $_GET['id']);
    $module = $module !== NULL ? $module : $identifier;
} else {
    $_GET['id'] = $identifier = $module = 'jameson_read';
}

优点：网址无plugin.php尾巴，手机版输入网址后直接显示插件页面
缺点：需要有权限操作服务器，更改服务器配置。网站后台-界面-导航 中首页设置将失效，无论如何设置，网站首页始终会是插件

计划任务

为降低系统消耗提升性能，图书分类和图书推荐以及排行榜数据通过计划任务进行更新，每小时执行一次

后台-工具-计划任务-小说阅读器数据更新，每当你进行推荐更新和分类修改后，需要手动到此位置执行此计划任务

常见问题

  1. 为什么更新或安装后看不到图书数据

    因为第一次计划任务还没有执行，解决方法是:到计划任务处手动执行 小说阅读器数据更新 这条计划任务
